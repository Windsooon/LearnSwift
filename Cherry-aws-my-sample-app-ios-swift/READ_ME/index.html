My Sample App

This download package contains an Xcode project for a fully functioning Mobile App, which uses your configured AWS resources and identifiers, to demonstrate the project features you've selected. You can use this project as a starting point for your own product development, or you can follow the instructions below to copy and paste the required code into your own product.

IMPORTANT: This package was custom generated for you, using your selected project options. It contains your own resource and account identifiers and should be kept private.

        Your system must include the following:
          - Xcode 7.1 or newer

      • 2. Install Xcode


        If you have not already installed the Xcode IDE (Integrated Development Environment) on your host, you can do download it from the following location:
        https://developer.apple.com/xcode/download


    • Facebook Sign-in

      • 3. Add iOS Platform to Facebook App


        In order for Facebook Sign-in to work properly, you will need to add the iOS platform to your Facebook App in the Facebook developer console. Log in to the Facebook developer console and select your app under "My Apps" menu option. Select "Settings" from the left menu. Scroll down and click on the "+ Add Platform" button and select iOS. Set the following field:

          Bundle ID = com.amazon.MySampleApp

        When you move beyond the sample app, you will set this to match your own project, but for now this values will work to demonstrate Facebook Sign-in, in the sample app.

  • Compile and Run
    • Launch App

      • 4. Open MySampleApp Project in Xcode


        Once you have downloaded and unzipped the sample app project files, please open Xcode and choose...

        Open another project...


        Find the folder where you unzipped the zip file and select the "MySampleApp/MySampleApp.xcodeproj" file. Xcode will open the project.

        Once Xcode finishes opening the project choose...

        Project -> Run


        Xcode will build and launch the app on a device or on the iOS Simulator.

    • User Identity - Try It!

      • 6. Demonstrate User Identity


        Launch MySampleApp on a device or on the iOS Simulator. If you've selected to enable User Sign-in as required in your project, the sign-in page will launch and you should sign in to the app. If you've selected to enable User Sign-in as optional, tap the "Sign-In" button on the top right to launch the sign-in page. Sign-in to one of the providers you have set up and it should take you back to the main screen and the "Sign-In" button will change to "Sign-Out". Then, tap on the "User Sign-in" demonstration section of the App and tap on the "Demo User Identity" button at the bottom. You will see a unique identifier for the user as well as a name for the user. This unique identifier identifies the user to AWS systems and provides the user temporary permissions to access your configured AWS resources.

      • 7. Verify User Identity


        Find the Cognito Identity Pool that your project is using. You can check which identity pool your project is using by going to the "Resources" section in the Mobile Hub console. The name of the identity pool should match the name of your project appended by "MOBILEHUB".

        Go to the AWS Cognito console. Click on the Cognito Identity Pool for your project and you will see that the "Total identities" value is now greater than zero. If you used Facebook or Google Login then you will see the number of authenticated users for that provider under the "Authentication methods" in the Cognito console.
        https://console.aws.amazon.com/cognito/


    • NoSQL Database - Try It!

      • 38. Demonstrate NoSQL Database


        Launch MySampleApp in the iOS simulator and click on the "NoSQL Database" section of the app and then click "Demo NoSQL Database" at the bottom of the next screen. You will see a list of the DynamoDB tables that were added to your project. Tap on the table that you would like to use in the demo.

        A list of example operations to view data in your table will now be shown. Tap the "Insert Sample Data" button in the bottom left of the screen to add random sample data to your table. Click on any of the list items to demonstrate running get, query, and scan operations on your table. When you are done, click on "Remove Sample Data" in the bottom right to remove the data that was inserted by the demo.



      • 39. Verify Data in DynamoDB Console


        Go to the Amazon DynamoDB console. You can click the table name in the "Resources" panel within Mobile Hub to navigate directly to your table in the Amazon DynamoDB console. Once you have inserted sample data into one of your tables, you will see it reflected in the DynamoDB console under the "items" tab for your table. Also, any entries you add to your table from the console will be reflected in the app when you use the Scan without Filter operation in the sample app.

      • 9. Identity Manager - Identifies the User


        This section simply provides information about the generated code in the MySampleApp project.

        The AWS Identity Manager keeps track of the identity of the current app user, by storing the user's Amazon Cognito Identity ID. In order to do this, the identity manager handles some of the App Delegate calls from the iOS platform, and it is partially responsible for signing the user in to the app, regardless of whether a third party identity provider, such as Facebook or Google, is in use.
        MySampleApp/MySampleApp/AmazonAws/AWSMobileHubHelper.framework/Headers/AWSIdentityManager.h

      • 10. NoSQL Demo Code - Demonstrate Amazon DynamoDB Table Operations


        This section simply provides information about the generated code in the MySampleApp project.

        Your generated Sample App contains a demonstration of using the DynamoDB Mapper to perform queries and scans against the NoSQL tables you configured in your Mobile Hub project. The following folder contains all the code for the NoSQL Demo. The *Table.swift files contain the example code to demonstrate insert, remove, query, and scan operations against your tables. The files named .swift is the model class for your configured table. The DynamoDB object mapper uses these annotated model classes to provide convenient marshaling between your Swift App and DynamoDB's JSON requests and responses.
        MySampleApp/MySampleApp/Demo/NoSQL

    • Development Methods

      • 13. Starter versus Example


        There are two convenient ways you can use what is provided in MySampleApp to enable your own app project with your configured features and cloud resources. You can use MySampleApp as a starter app, and build your own app on top of it. Alternatively, you can use MySampleApp as an example, and copy and paste the source code from MySampleApp into your own app project.

    • Use as a Starter App

      • 14. Use MySampleApp as a Starter App


        When you use MySampleApp as a starter app for your own app project, it serves as a starting point for your own App development. Remove the "demo" features, replace the app name and package name, build in your own app logic and user interface, and you're done!

      • 15. Remove the "demo" features


        You may delete the "Demo" folder, which contains the examples demonstrating the features you've enabled in your project. You might want to keep snippets of the examples to use in your own code. You should also modify the Main.storyboard file with your application's own user interface and modify the SplashViewController and MainViewController, as you see fit for your application.

        Please copy this folder and all of its sub-folders from the MySampleApp project into your project.

      • 23. Add a Run Script phase to your project


        After adding the AWS SDKs as Embed Frameworks, under the Build Phases tab in your Target, click the + button on the top left and then select New Run Script Phase. Add the following line in the text box:

        bash "${BUILT_PRODUCTS_DIR}/${FRAMEWORKS_FOLDER_PATH}/AWSCore.framework/strip-frameworks.sh"

        This step is required for the app to be published on the iTunes App Store.

    • Facebook Sign-In Issues

      • 36. Facebook "App Not Setup" error message


        If you are getting the message "App Not Setup" after a successful login it means that your Facebook App is still in development mode. For testing purposes, you can add test users to your Facebook App via the Facebook Developer Console.
        To do that create a test user for your Facebook development application:
        1. Go to https://developers.facebook.com, log in with your developer credentials.
        2. From the "My Apps" menu, select your application used in the sample app.
        3. On the left pane, select "Roles".
        4. Under "Testers", click "Add Testers".
        5. In the Add Testers dialog, add the username of the test user (i.e. https://www.facebook.com/myuser.name.50).
        6. Click Submit.
        Now you can have that test user log into your sample app.

        For more information on adding test users go to: https://developers.facebook.com/docs/apps/test-users.

        If your app is "Live" to the General Public, then you need to provide a valid Contact Email address for your Facebook App, go to the Facebook developer portal and go to the "Settings" section of your app. Enter a valid email under "Contact Email" and click "Save changes".

      • 37. Facebook "Given URL is not allowed..." error message


        You attempt to sign in and you see this error message:

        Given URL is not allowed by the Application configuration: One or more of the given URLs is not allowed by the App's settings. To use this URL you must add a valid native platform in your App's settings.

        This indicates the iOS platform has not been added to your Facebook app. Make sure you follow the steps in the "Setup - Facebook Sign-in" section of this page.
